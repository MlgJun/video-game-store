Гайд на гит:
1) При добавлении нового функционал создаешь новую ветку git checkout -b <название_ветки>
2) Пишешь код, коммитишь(чем меньше коммит, тем лучше) и как все закончил - git push origin <название_ветки>
3) Переходишь в репозиторий жмакаешь по create pull request -> пишешь коммент -> ждешь одобрения pull request'а
4) Если pull request не прошел, то делаешь нужные изменения, также пушишь, создаешь pull request
5) Если все гуд, то после pull request'а - git checkout main -> git pull origin main

Гайд на миграции:
1) Обновляешь бд Update-Database(первый раз)
2) Создаешь миграцию Add-Migration <название_миграции>
3) Обновляешь бд Update-Database
4) Не забудь закоммитить миграции
